<!-- artifact:quote:start -->
> - Implemented texture streaming, now many textures are loaded and unloaded on demand dynamically to reduce VRAM usage especially in higher graphics qualities. About a 50% reduction in VRAM usage from ~6 GB to ~3 GB. This can be toggled off/on in the Graphics settings menu. Click to expand...
<!-- artifact:quote:end -->
So when we're running the game with texture streaming enabled we are seeing some effect on rendertimes due to the overhead of checking and streaming in new textures. So I would say that if you're running into framerate drops then you might want to try to turn texture streaming off to see if that helps with any bottlenecks for a steady framerate. As for how it plays out on integrated graphics specifically we haven't made extensive comparisons so my musings here are my best guesses. Integrated graphics might be somewhat more affected for the write time to the RAM when we stream in textures - however that's only part of the operations we do on the memory and having a lower footprint will also mean that you will contend with less space in your RAM with other applications and thus might see an improvement in throughput in cases you are running high on your RAM usage. However for the systems we've checked we can notice the overhead on some systems but the slowdown has not been significant. What is significant however is the large amount of VRAM that we're saving. This is partly due to the nature of our game where we have a lot of textures and assets that make up our character portraits but we will only ever show a tiny portion of all those assets on the screen at any single time. Without texture streaming the game has to keep in memory the high resolution textures for a low-born character in Portugal even if you're playing and only view characters down in India. With texture streaming we can dynamically stream in the higher resolution textures based on what you're trying to view on the screen. This significantly reduces the active memory we need to handle for our textures. This is theoretically good for loadtimes but more importantly it allows the game to play much better on lower-end GPUs that were running close to the VRAM limit. The texture streaming technology also allows us to future-proof our systems since we still want to add a lot more variety in clothing and props for our characters going forward with DLCs in the years ahead. And over time that would increase our VRAM memory footprint to require more and more high-end GPUs to even run the game - which would not be a desirable outcome.

> Divine said: So when we're running the game with texture streaming enabled we are seeing some effect on rendertimes due to the overhead of checking and streaming in new textures. So I would say that if you're running into framerate drops then you might want to try to turn texture streaming off to see if that helps with any bottlenecks for a steady framerate. As for how it plays out on integrated graphics specifically we haven't made extensive comparisons so my musings here are my best guesses. Integrated graphics might be somewhat more affected for the write time to the RAM when we stream in textures - however that's only part of the operations we do on the memory and having a lower footprint will also mean that you will contend with less space in your RAM with other applications and thus might see an improvement in throughput in cases you are running high on your RAM usage. However for the systems we've checked we can notice the overhead on some systems but the slowdown has not been significant. What is significant however is the large amount of VRAM that we're saving. This is partly due to the nature of our game where we have a lot of textures and assets that make up our character portraits but we will only ever show a tiny portion of all those assets on the screen at any single time. Without texture streaming the game has to keep in memory the high resolution textures for a low-born character in Portugal even if you're playing and only view characters down in India. With texture streaming we can dynamically stream in the higher resolution textures based on what you're trying to view on the screen. This significantly reduces the active memory we need to handle for our textures. This is theoretically good for loadtimes but more importantly it allows the game to play much better on lower-end GPUs that were running close to the VRAM limit. The texture streaming technology also allows us to future-proof our systems since we still want to add a lot more variety in clothing and props for our characters going forward with DLCs in the years ahead.
